Redis数据类型总结

2019-04-16 Redis

主要记录一下Redis数据库(以key,value形式存储数据)中的五种数据类型：string(字符串)、hash(哈希）、list(列表）、set(集合)、zset(有序集合)；会周期性的把更新的数据写入到磁盘或者把修改操作写入追加的记录文件。

String类型

  • 字符串类型是Redis中最为基础的数据存储类型，是二进制安全的(该类型可以接受任何格式的数据)，是标准的key-value。
  • 应用场景：统计网站访问数量、当前在线人数、接口防刷、缓存。

散列hash

  • Redis中的散列可以看成具有String key和String value的map容器，可以将多个key-value存储到一个key中。
  • 应用场景：如存储、读取、修改用户属性（name, age, pwd等）

列表list

  • 允许用户从序列的两端推入或者弹出元素，是由多个字符串组成的有序可重复的序列（链表结构），所以向两端添加、删除元素的时间复杂度为O(1)。
  • 应用场景：最新消息排行榜、消息队列(以完成多程序之间的消息交换)

集合set

  • Redis的集合是无序不可重复的。在执行插入、删除以及判断是否存在某元素时，效率很高。最大的优势在于可以进行交集并集差集操作。
  • 应用场景：利用交集求共同好友、利用唯一性统计访问网站的所有独立IP、好友推荐的时候根据tag求交集等

有序集合sorted set

  • 字符串集合，不允许有重复成员。有序集合中每一个成员都会有一个分数(score)与之关联，Redis正是通过分数来为集合中的成员进行从小到大的排序。尽管有序集合中的成员必须是唯一的，但是分数(score)却可以重复。
  • 应用场景：各种排行榜(游戏积分排行榜等)、展示用户信息及用户分数(例如学校中按学生分数排序）

最后，还有个对key的通用操作，所有的数据类型都可以使用的
